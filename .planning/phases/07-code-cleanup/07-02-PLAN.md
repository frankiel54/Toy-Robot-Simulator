  <name>Task 1: Fix correctness bug — reject Unset direction in TryParsePlaceArgs (Correctness)</name>
  <files>ToyRobot/CommandParser.cs</files>
  <action>
    In CommandParser.TryParsePlaceArgs, the current implementation calls Enum.TryParse and returns true for any successfully parsed Direction member, including Direction.Unset. This means "PLACE 1,2,UNSET" is silently accepted as a valid placement.

    After the Enum.TryParse call succeeds, add an explicit guard: if the parsed direction equals Direction.Unset, reset the out parameters x and y to 0, reset direction to default, and return false. This makes the Unset member behave like an unparseable string.

    The guard belongs immediately after the block where Enum.TryParse succeeds and before the method's true return path. The existing method structure uses a chained boolean expression; refactor as needed to insert the guard cleanly — a two-step approach (parse into a local, then guard) works well here.
  </action>
