<objective>
Convert the core domain types to a cleaner, sentinel-free model.

Purpose: Eliminate two design liabilities introduced in earlier phases — the Direction.Unset sentinel value (a boolean flag disguised as an enum member) and the mutable Robot class (which required callers to track placement state via a separate bool). Making Robot an immutable record and Simulator's robot field nullable removes both liabilities at once, and dropping the raw X/Y/Facing pass-through properties from Simulator forces consumers to go through the Report() contract.

Output:
- ToyRobot/Direction.cs — enum with four members only; wildcard arms in extension methods throw instead of silently returning
- ToyRobot/Robot.cs — immutable record with primary constructor; GetNextPosition() is the only instance method
- ToyRobot/Simulator.cs — Robot? field; Place/TurnLeft/TurnRight/MoveForward use record construction and with-expressions; no X/Y/Facing properties
- ToyRobot/CommandParser.cs — TryParsePlaceArgs has no sentinel guard block

Decision traceability: REQ-08-01 (Robot record), REQ-08-02 (remove Unset), REQ-08-03 (remove X/Y/Facing)
</objective>

  (b) In DirectionExtensions.TurnLeft, replace the wildcard arm that currently returns the unchanged direction value with a wildcard arm that throws ArgumentOutOfRangeException, passing nameof(direction) as the parameter name.   (b) Do not write a parameterless constructor.
  (c) Do not write a three-argument constructor body — the primary constructor IS the constructor.
  (d) Do not write PlaceAt, MoveTo, TurnLeft, or TurnRight as instance methods.
  (e) Retain GetNextPosition as a public instance method returning (int x, int y). It reads the XPos, YPos, and Direction properties that the primary constructor generates. Update its wildcard arm from the current "return same position" fallback to throw ArgumentOutOfRangeException(nameof(Direction)).

  </action>

  TurnLeft method: Replace the call to Robot.TurnLeft() with a null-conditional with-expression. The body must guard on _robot being non-null before mutating: if (_robot is not null) _robot = _robot with { Direction = _robot.Direction.TurnLeft() };

  TurnRight method: Apply the identical pattern using TurnRight() on the Direction.

  MoveForward method: Add a null guard at the top returning false when _robot is null. Replace the call to Robot.GetNextPosition() with _robot.GetNextPosition(). Replace the call to Robot.MoveTo(x, y) with a with-expression: _robot = _robot with { XPos = x, YPos = y };

  Report method: Replace the check "if (!RobotPlaced)" with "if (_robot is null)". Replace the interpolation that reads Robot.XPos, Robot.YPos, Robot.Direction with _robot.XPos, _robot.YPos, _robot.Direction.

  IsPlaced property: Replace the body "RobotPlaced" with "_robot is not null".

  Remove these three public properties in their entirety — they must not appear anywhere in the file:
  - public int X => Robot.XPos;
  - public int Y => Robot.YPos;
  - public Direction Facing => Robot.Direction;

After the compound if/return false block that validates the three-part parse (the block currently ending at "return false;"), there is a second guard block of approximately four lines that compares the parsed direction to a sentinel enum member and resets x, y, direction before returning false. Delete that entire guard block. The method must return true immediately after the compound parse succeeds, with no further direction-value checking.
